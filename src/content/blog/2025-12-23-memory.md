A reflection on work done while at Succinct Labs.

## The problem

Succinct's main product is a RISC-V zkVM. At a high level, you're supposed to be able to execute any RISC-V program with some assertions about its properties, obtain a small proof, and later cheaply verify that those properties hold. This involves a lot of computationally expensive fancy cryptography, in particular with a huge memory footprint. I couldn't find a lot of good resources for how to deal with this kind of stuff online, so I'm compiling some stuff I've learned here now.

## 1. Move stuff to device

A lot of the cryptography we need to do is very GPU friendly. Moving stuff to and from the GPU is expensive. So one way to save memory on the host is to move data to device, and manage it manually whenever possible.

For each RISC-V cycle we execute, we need to generate a bunch of extra information about it that is needed for proving. A compressed form of this extra proving data is called a `Record`, and it's expanded into an algebraic form called a `Trace`. Generating records from RISC-V instructions is not very GPU-friendly, but generating traces from records is. So one big optimization we found was to move our trace generation to device.
